# FI Quotation Web App v1.3

ระบบสร้างและจัดการใบเสนอราคา ด้วย GitHub Pages + Supabase

## สิ่งที่เพิ่มใน v1.3

- เพิ่มสถานะเอกสาร `ชำระเงิน` และปรับชื่อสถานะทั้งหมดเป็นภาษาไทยแบบกระชับ
- เพิ่ม `sent_at`, `paid_at`, และ `cancelled_reason`
- ตอนเปลี่ยนเป็น `ส่งแล้ว` หรือ `ชำระเงิน` ให้ระบุวันที่ได้ โดยค่าเริ่มต้นเป็นวันนี้
- ตอนยกเลิกเอกสารสามารถกรอกเหตุผลได้แบบ optional
- หน้า List View เพิ่ม checkbox เลือกหลายรายการ และ Bulk Update สถานะได้
- Bulk Update รองรับ `ส่งแล้ว`, `ชำระเงิน`, `ยกเลิก`
- เพิ่ม filter ฝ่ายขายแบบ dropdown โดยรวมทุก user ที่เคยเป็น owner ของใบเสนอราคา
- เพิ่ม sorting ที่หัวคอลัมน์หลักทุกคอลัมน์
- ปรับ Date Range UI ใหม่ตามรูปแบบ business app พร้อม preset เดือนนี้ / เดือนที่แล้ว / ปีนี้ / ปีที่แล้ว
- จำกัดช่วงวันที่เสนอราคาและช่วงวันหมดอายุไม่เกิน 3 เดือนต่อช่วง
- Dashboard เพิ่มยอดส่งแล้วเดือนนี้ และยอดชำระเงินเดือนนี้
- Dashboard เพิ่มกราฟ 12 เดือน เปรียบเทียบยอดส่งแล้วกับยอดชำระเงิน
- Dashboard เพิ่มกราฟจำนวนใบเสนอราคาที่ส่งแล้ว แยกตามฝ่ายขายและเดือน
- แก้ reliability เพิ่มเติมเมื่อเปลี่ยน tab แล้วกลับมาใช้งานต่อ
- ตัดสัญลักษณ์ `฿` ออกจากทุกจุดในระบบ เหลือเฉพาะตัวเลข
- ตัดชื่อผู้ใช้ที่ซ้ำข้างปุ่มออกจากระบบออก เหลือเฉพาะ user chip ด้านบน

## วิธีติดตั้ง

1. Copy `index.html`, `style.css`, `script.js` ไปทับใน repo `fi-quotation-web`
2. Copy โฟลเดอร์ `supabase` ไปไว้ใน repo
3. ไปที่ Supabase SQL Editor แล้วรัน `supabase/patch_v1_3.sql`
4. เปิด Live Server ทดสอบก่อน push
5. Commit และ Push ขึ้น GitHub Pages

## Push ผ่าน VS Code Terminal

```bash
git status
git add .
git commit -m "Release v1.3 status analytics and bulk actions"
git push origin main
```

## ทดสอบหลักหลังติดตั้ง

- Refresh หน้าเว็บแล้วไม่เห็นหน้า Login แวบขึ้นมาถ้ามี session อยู่
- เปลี่ยนไป tab อื่นแล้วกลับมา เมนูและปุ่มยังใช้งานได้โดยไม่ต้อง refresh
- สถานะในระบบแสดงเป็นภาษาไทยทั้งหมด
- Confirm ใบเสนอราคาแล้วยังออกเลข QTN ได้ปกติ
- เปลี่ยนสถานะจาก `ยืนยัน` เป็น `ส่งแล้ว` พร้อมระบุวันที่ส่งได้
- เปลี่ยนสถานะจาก `ส่งแล้ว` เป็น `ชำระเงิน` พร้อมระบุวันที่ชำระเงินได้
- ยกเลิกเอกสารพร้อมกรอกเหตุผลได้
- หน้า List View เลือกหลายใบแล้ว Bulk Update สถานะได้
- หน้า List View filter ฝ่ายขายได้
- หน้า List View sorting ที่หัวคอลัมน์ทำงาน
- Date Range UI เลือก preset และช่วงวันที่เองได้
- ช่วงวันที่เกิน 3 เดือนต้องไม่อนุญาตให้ Export
- Export Excel เป็น `.xlsx` และไม่มีสัญลักษณ์ `฿`
- Dashboard แสดงยอดส่งแล้ว / ยอดชำระเงิน / กราฟ 12 เดือนถูกต้อง

## Security Notes

- ใช้เฉพาะ Supabase anon public key ใน frontend
- ห้ามใส่ service_role key ใน `script.js`
- การเปลี่ยนสถานะสำคัญทำผ่าน RPC `change_quotation_status_v13`
- Sales เห็นและแก้เฉพาะข้อมูลของตัวเองตาม RLS
